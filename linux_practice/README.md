# Практика по модулю Linux

Курс **«Аналитик данных»** · платформа [simulative.ru](https://app.simulative.ru/course/65/2428)

Bash-скрипт, последовательно выполняющий 13 задач практики и закрепляющий работу с переменными, перенаправлением потоков, правами, функциями, аргументами командной строки и сетевыми запросами.

## Структура репозитория

```
linux-practice/
├── practice.sh   # основной скрипт со всеми задачами
├── README.md     # этот файл
└── .gitignore    # исключает логи и временные файлы
```

## Содержание практики

| №  | Тема                                 | Команды / конструкции                           |
|----|--------------------------------------|-------------------------------------------------|
| 0  | Завершение при ошибке                | `set -e`                                        |
| 1  | Переменная путей                     | `export`, `basename`                            |
| 2  | Информация об ОС                     | `cat /etc/os-release`, `grep -i`                |
| 3  | Обновление системы (root)            | `$EUID`, `apt update && apt upgrade`            |
| 4  | Временный alias                      | `shopt -s expand_aliases`, `alias`              |
| 5  | Файл лога с перезаписью              | `rm`, `touch`                                   |
| 6  | Список файлов в лог                  | `ls >> logfile.log`                             |
| 7  | Подсчёт строк                        | `wc -l`                                         |
| 8  | Переменные окружения                 | `$HOME`, `$PATH`                                |
| 9  | Функции                              | `my_function()`                                 |
| 10 | Аргументы командной строки           | `$1`, сравнение `--info`                        |
| 11 | Права на файл                        | `chmod 777`                                     |
| 12 | ТОП-3 процессов по памяти            | `ps aux --sort=-%mem | head -4`                 |
| 13 | Простой HTTP-запрос                  | `curl`                                          |

## Запуск

```bash
# Обычный запуск (выполнение всех задач)
bash practice.sh

# Запуск с аргументом --info — вызывает демонстрационную функцию
bash practice.sh --info

# Сделать скрипт исполняемым и запустить
chmod +x practice.sh
./practice.sh
```

> **Замечание по `set -e`.** В терминале выполнение несуществующей команды прервет сессию, а в скрипте (`bash practice.sh`) — лишь завершит сам дочерний процесс, не закрывая терминал. Скрипт запускается в дочерней оболочке.

> **Задача 3 (обновление системы).** Запуск от `root` выполнит `apt update && apt upgrade`. Запуск от обычного пользователя выведет сообщение и пропустит обновление.

> **Задача 4 (alias).** Для применения алиаса в текущей (родительской) сессии скрипт можно запустить через `source`:
> ```bash
> source ./practice.sh
> ```

## Сдача работы

Архив для сдачи собирается командой:

```bash
zip -r linux-practice.zip practice.sh README.md .gitignore
```

Полученный `linux-practice.zip` загружается на страницу задания или прикрепляется ссылкой на репозиторий.
